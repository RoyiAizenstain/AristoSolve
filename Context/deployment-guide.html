Full-Stack Deployment Guide

React + Node.js + MySQL → AWS RDS + Render

1
Start AWS RDS Database Creation
Takes ~5 minutes — kick it off first, then move on

Create the instance

  • Go to AWS Console → search RDS → click Create database
  • Choose: Standard create → MySQL → Free tier template
  • Set Master username: admin
  • Set a Master password and save it somewhere
  • Under Connectivity → Public access: Yes
  • Click Create database and move on to Step 2 immediately — don't wait
While AWS creates the database (~5 min)
Step 2A — Build & Push React App
  • React JSX must be compiled before it runs in the browser
  • Run Parcel to compile everything into client/dist/
cd client
npm install
npm run build

cd ..
git add client/dist/
git commit -m "build client"
git push
What this does: Parcel reads your .jsx files, compiles JSX to plain JS, and bundles everything into 3 files in client/dist/. Express will serve these files in production.
Step 2B — Create Render Web Service
  • Go to render.com → New → Web Service
  • Connect your GitHub repository
  • Set Root Directory: server
  • Set Build Command: npm install
  • Set Start Command: node app.js
  • Add environment variables (see Step 3)
  • Click Deploy Web Service
Note: You can set DB_HOST to a placeholder for now and update it once RDS is ready. Render lets you edit env vars and redeploy at any time.
3
Render Environment Variables
Add these in the Render dashboard under Environment

Required variables

KeyValue
DB_HOSTyour RDS endpoint (add once available)
DB_USERadmin
DB_PASSWORDyour RDS password
DB_NAMEevent_map
JWT_SECRETany long random string
GEMINI_API_KEYyour Google AI Studio key
GEMINI_MODELgemini-2.5-flash
Never commit secrets to GitHub. Always set sensitive values through Render's environment variables panel — never in code.
4
Finish AWS Setup
Once the RDS status shows "Available"

Get the endpoint

  • Click on your RDS instance → Connectivity & security tab
  • Copy the Endpoint (e.g. xxx.rds.amazonaws.com)
  • Paste it as DB_HOST in Render's environment variables

Open firewall access

  • In the instance page → click the Security Group link
  • Click Inbound rules → Edit inbound rules
  • Add rule: Type = MySQL/Aurora · Source = Anywhere (0.0.0.0/0)
  • Click Save rules

Seed the database

  • Update server/.env with the new DB_HOST and DB_PASSWORD
  • Run the seed script — creates all tables and mock data in one command
cd server
npm run seed
Expected output: "Done. Database "event_map" recreated with all tables and mock data."
Test accounts (password: password123): alice · bob · charlie
5
Trigger Final Deploy
Once DB_HOST is set in Render and the database is seeded

Redeploy on Render

  • In Render dashboard → click Manual Deploy → Deploy latest commit
  • Wait for the build log to show "Your service is live"
  • Open the .onrender.com URL — the app is live and connected to the database
6
Update & Redeploy After Changes
Every time you modify the code

Client (React) change

  • Edit files in client/src/, then rebuild and push
cd client && npm run build
cd ..
git add client/dist/
git commit -m "rebuild client"
git push

Server (Node.js) change

  • Edit files in server/, then push directly
git add .
git commit -m "update server"
git push
Render redeploys automatically on every push to the main branch. No manual action needed.
