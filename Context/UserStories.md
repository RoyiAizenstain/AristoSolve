# User Stories — AristoSolve (tracked)

Legend: ✅ done · 🔲 Phase 2 · ❌ not started

---

## 1. User Management & Authentication

- ✅ As a new user, I want to register with name, email, and password
- ✅ As a registered user, I want to log in securely
- ✅ As a user, I want the system to identify my role (Candidate / Company / Admin) and show relevant screens
- ✅ As a user, I want to update my profile settings (display name, email, theme, notifications)

---

## 2. Recruitment Track (Company-Led Testing)

- ✅ As a candidate, I want to see a list of problems on my dashboard
- ✅ As a candidate, I want to tackle algorithm / system-design / debugging problems
- ✅ As a candidate, I want to choose my preferred language (Python / Java / JavaScript)
- ✅ As a candidate, I want to interact with an AI mentor (AristoBot) — mocked replies Phase 1
- 🔲 As a candidate, I want a built-in code editor with syntax highlighting (Monaco — Phase 2)
- 🔲 As a candidate, I want to run code against test cases and see real output (Piston — Phase 2)
- ❌ As a candidate, I want to view my evaluation report after company review
- ❌ As a candidate, I want to review past conversations and submissions
- ✅ As a company rep, I want to create problems with a custom evaluation prompt
- ✅ As a company rep, I want to see candidate scores and evaluation reports on my dashboard

---

## 3. Self-Learning Track

- ✅ As a student, I want to access a problem repository categorized by difficulty and topic
- ✅ As a student, I want to filter problems by difficulty and topic
- ✅ As a user, I want an AI mentor chat that guides thinking without giving the answer (mocked)
- 🔲 As a user, I want to view my progress statistics over time (Progress page — Phase 2)

---

## 4. System Operations & Control

- ✅ As an admin, I want every request logged (method, URL, time)
- ✅ As an admin, I want mandatory field validation on all API inputs
- ✅ As an admin, I want role-based access control enforced
- ✅ As an admin, I want to create, edit, and delete users
